# ADR-002 账本作为计费真相源

## 状态

Accepted

## 背景

历史系统中，余额、配额、订阅使用量往往分散在多张业务表和临时逻辑中，容易出现重复扣费、漏退款、手工重置不可审计、报表与真实扣费不一致的问题。

## 决策

CodeGo v2 的计费真相源统一收敛到账本体系：

- `billing.accounts`
- `billing.ledger_entries`
- `billing.reservations`
- `billing.settlements`
- `billing.refunds`

余额、订阅配额和人工调整必须通过账本动作体现，读模型只做投影，不作为真相源。

## 后果

正面：

- 所有计费动作可审计
- 能支持幂等、对账、回放和补偿
- 管理员手动重置订阅配额可纳入统一账本语义

负面：

- 初期开发复杂度上升
- 所有旧逻辑都要逐步迁移

## 落地约束

- 禁止直接修改“余额结果字段”绕过账本
- settlement/refund/release 都必须具备幂等键
- 所有高风险手工操作必须生成审计日志
